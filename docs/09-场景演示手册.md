## 场景一：新设备从接入到故障预警

> 业务背景：工厂运维团队需要实时监控设备健康状况，在故障发生前安排预防性维护。

### 步骤 1：查看设备健康总览

发现哪些设备处于危险状态。

```bash
curl http://localhost:8000/api/v1/maintenance/equipment-health
```

**关注点**：返回的 `summary` 中 `critical: 2` 表示有 2 台设备处于严重状态。

```json
{
  "summary": {
    "total": 15,
    "healthy": 9,
    "warning": 4,
    "critical": 2,
    "avg_score": 79.7
  },
  "equipment": [
    {"id": 15, "name": "空压机-阿特拉斯", "health_score": 38.9, "risk_level": "critical"},
    {"id": 10, "name": "磨床-上海机床",   "health_score": 45.2, "risk_level": "critical"},
    {"id": 9,  "name": "数控车床-沈阳机床", "health_score": 68.5, "risk_level": "warning"},
    ...
  ]
}
```

### 步骤 2：查看高风险设备详情

深入分析空压机（ID=15，评分 38.9）的健康趋势。

```bash
curl http://localhost:8000/api/v1/maintenance/equipment/15/health
```

```json
{
  "id": 15,
  "name": "空压机-阿特拉斯",
  "model": "GA37+",
  "health_score": 38.9,
  "risk_level": "critical",
  "trend": [
    {"date": "2026-04-15", "health_score": 52.1},
    {"date": "2026-04-16", "health_score": 48.7},
    ...
    {"date": "2026-04-21", "health_score": 38.9}
  ],
  "breakdown": {
    "vibration": 65.2,
    "temperature": 72.1,
    "pressure": 58.3,
    "electrical": 81.0,
    "wear": 42.5
  },
  "recommendation": "设备存在严重风险，建议立即停机检修！"
}
```

**解读**：
- 健康评分持续下降（52.1 → 38.9），趋势恶化
- `wear`（磨损）维度仅 42.5 分，是主要短板
- 系统建议：立即停机检修

### 步骤 3：查看故障预测

确认故障类型和预计时间窗口。

```bash
curl http://localhost:8000/api/v1/maintenance/predictions
```

```json
{
  "data": [
    {
      "equipment_id": 15,
      "equipment_name": "空压机-阿特拉斯",
      "fault_probability": 0.611,
      "predicted_fault": "轴承磨损",
      "estimated_days": 2,
      "risk_level": "critical"
    },
    {
      "equipment_id": 10,
      "equipment_name": "磨床-上海机床",
      "fault_probability": 0.548,
      "predicted_fault": "轴承磨损",
      "estimated_days": 3,
      "risk_level": "critical"
    },
    ...
  ]
}
```

**解读**：空压机 61.1% 概率在 2 天内发生轴承磨损故障，需立即行动。

### 步骤 4：查看传感器数据验证

拉取温度传感器数据，确认异常趋势。

```bash
curl "http://localhost:8000/api/v1/quality/spc/temperature?equipment_id=15&hours=24"
```

```json
{
  "parameter": "temperature",
  "data": [
    {"timestamp": "2026-04-21T08:00:00", "value": 68.5, "ucl": 70.0, "lcl": 60.0, "out_of_control": false},
    {"timestamp": "2026-04-21T10:30:00", "value": 71.2, "ucl": 70.0, "lcl": 60.0, "out_of_control": true},
    ...
  ],
  "cpk": 0.82,
  "equipment_id": 15
}
```

**解读**：Cpk=0.82 < 1.33（标准值），说明过程能力不足，温度波动已超出控制限。

### 步骤 5：创建维修工单

```bash
curl -X POST http://localhost:8000/api/v1/maintenance/work-orders \
  -H "Content-Type: application/json" \
  -d '{"equipment_id": 15, "type": "emergency", "priority": "critical", "description": "轴承磨损预警，需立即更换轴承"}'
```

```json
{"id": 147, "status": "pending", "message": "维修工单已创建"}
```

### 步骤 6：AI 助手确认决策

```bash
curl -X POST http://localhost:8000/api/v1/ai/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "空压机需要维护吗？"}'
```

```json
{
  "intent": "equipment",
  "response": "有 6 台设备需要关注：\n- 空压机-阿特拉斯: 健康评分 38.9\n- 磨床-上海机床: 健康评分 45.2\n..."
}
```

### 场景一总结

```
设备健康总览 → 发现异常设备 → 查看详情+趋势 → 故障预测 → 传感器验证 → 创建工单 → AI确认
      ↓              ↓              ↓              ↓            ↓            ↓          ↓
  critical×2    空压机38.9分    磨损42.5分     61.1%概率     Cpk=0.82    工单#147    6台需关注
```

## 场景二：质量异常根因分析与追溯

> 业务背景：检验员发现一批产品有尺寸超差缺陷，需要追溯根因并确定影响范围。

### 步骤 1：查看 SPC 控制图异常

```bash
curl http://localhost:8000/api/v1/quality/spc/dimension
```

```json

  "parameter": "dimension",
  "data": [
    {"value": 50.035, "ucl": 50.020, "lcl": 49.980, "out_of_control": true},
    {"value": 50.028, "ucl": 50.020, "lcl": 49.980, "out_of_control": true},
    {"value": 50.015, "ucl": 50.020, "lcl": 49.980, "out_of_control": false},
```

```bash

```

```json
{
  "data": [
    {"defect_type": "尺寸超差",   "count": 12, "percentage": 30.0, "cumulative_percentage": 30.0},
```

**解读**：追溯链显示产品经历了 7 个环节，每步都合格。说明缺陷可能来自特定设备或批次。

### 步骤 5：通过图谱查看关联

查看设备 ID=4（DMG MORI）的关联关系，确认是否与其他缺陷有关联。


### 场景二总结

```
SPC异常 → 缺陷列表 → 帕累托找主因 → 追溯生产链 → 图谱关联 → 创建CAPA
   ↓          ↓           ↓              ↓             ↓           ↓
Cpk=0.95  尺寸超差×2  30%占比      7步追溯链     设备-工单关联  CAPA已创建
```
