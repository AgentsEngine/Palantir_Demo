## 1. 技术栈

- Python 3.11
- **FastAPI 0.115** — 路由 + 依赖注入 + OpenAPI
- **SQLAlchemy 2.0 (async)** — typed ORM；`Mapped[...]` 风格
- **Pydantic 2.x** + `pydantic-settings`
- **Alembic 1.14** — schema 迁移
- **asyncpg / aiosqlite** — 异步驱动
- **neo4j 5.x (async driver)** — 图数据
- **passlib[bcrypt] + python-jose** — 密码与 JWT
- **scipy / numpy / pandas** — 分析层

完整依赖见 `backend/requirements.txt`。

## 2. 启动与本地开发

最快路径（不依赖 Docker，使用 SQLite fallback）：

```bash
cd backend
python -m venv .venv && source .venv/bin/activate     # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp ../.env.example ../.env                            # 可选
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

## 3. 配置与环境变量

所有运行时配置都集中在 `app/config.py: Settings`，通过 `pydantic-settings` 从 `.env` 读取。

| 变量 | 默认 | 用途 |
|---|---|---|
| `SECRET_KEY` | `manufoundry-secret-key-change-in-production` | JWT 签发密钥（**生产必改**） |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | `480` | token 有效期（分钟） |
| `DEMO_AUTH_OPTIONAL` | `true` | 缺/坏 token 时是否退化为 guest（生产应 `false`） |
| `CORS_ORIGINS` | `[localhost:3000, ...]` | 允许的前端域；含 `*` 时自动禁用 credentials |
| `LOG_LEVEL` | `INFO` | logging 级别 |
| `POSTGRES_*` | localhost / manufoundry / 5432 | PG 连接 |
| `NEO4J_*` | bolt://localhost:7687 / neo4j | Neo4j |
| `REDIS_*` | localhost / 6379 / 0 | Redis |
| `OPENAI_API_KEY` | 空 | 可选 |

⚠ 不要在 router 内重新声明 `SECRET_KEY` 之类的常量。`app/api/auth.py` 之前的内置 SECRET 已经被移除并改为 `from app.core.security import create_access_token`，新代码必须沿用。

## 4. 日志规范

统一入口：

```python
from app.core.logging import get_logger
logger = get_logger(__name__)
```

- 启动时 `app/main.py` 调用 `setup_logging(settings.LOG_LEVEL)`，所有子模块继承同一格式：
  `2026-04-28 12:34:56 [INFO] app.api.dashboard: ...`
- **禁止使用 `print(...)` 与 bare `except: pass`**。后者会被 lint 拦截（noqa 仅在已记录 log 的 fallback 处允许）。
- 业务热路径用 `logger.warning(...)`；调试细节用 `logger.debug(...)`；未捕获异常由 `main.unhandled_exception_handler` 自动 `logger.exception(...)`。

## 5. 数据库会话与错误处理

### 5.1 三种获取 session 的方式

| 场景 | 推荐 API | 行为 |
|---|---|---|
| 路由依赖注入 | `db: AsyncSession = Depends(get_db)` | 失败抛 500，由全局 handler 接管 |
| service 内部 | `async with db_session() as s: ...` | 异常 rollback + log + 重新抛出 |
| router 兼容旧模式 | `await _try_db(_query)` | 异常 rollback + log，**返回 None**，调用方判 None 后 mock fallback |

### 5.2 router 兼容模式（当前主流）

每个 router 文件顶部统一：

```python
from app.core.db import safe_db_call as _try_db  # noqa: E402
```

调用：

```python
async def _query(db):
    result = await db.execute(select(Equipment))
    return {"data": [...]}

result = await _try_db(_query)
if result is not None:
    return result
return {"data": MOCK_EQUIPMENT}
```

`safe_db_call` 内部走 `db_session()`，因此异常的 rollback + log 与 router 自己写 try/except 等价。

### 5.3 何时不要用 `_try_db`

如果接口语义是「失败必须告警 5xx 而不是降级 mock」，直接用 `db_session()`：

```python
@router.post("/critical")
async def critical_op(body: X):
    async with db_session() as db:
        ...
        await db.commit()
        return {"ok": True}
```

这样异常会冒泡到 `unhandled_exception_handler`，前端拿到 500 而不是错误的 mock 数据。

## 6. Pydantic Schema 约定

- 输入 schema 用 `<Verb><Resource>` 命名（如 `LoginRequest`、`PageConfigCreate`）；
- 输出可以用 `dict` 直接返回（FastAPI 会自动 JSON 化），但**列表分页必须返回 `{"data": [...], "total": int, "page": int}`**，与前端 `services/api.ts` 解包模式 (`res.data?.data`) 对齐；
- 跨多个 router 共享的 schema 收纳到 `app/api/_<feature>_shared.py`（参考 `_model_driven_shared.py`）。`app/schemas/` 目录预留给将来按领域聚合时使用。

## 7. 新增一个业务路由

例：新增「工艺参数管理」`/api/v1/process-params`。

1. **建模**（如需）：在 `app/models/relational.py` 加 ORM 类。
2. **生成迁移**：`alembic revision --autogenerate -m "add process_params"`，复核 `alembic/versions/xxx.py`。
3. **建路由**：新建 `app/api/process_params.py`：

   ```python
   from fastapi import APIRouter, Depends
   from sqlalchemy import select
   from sqlalchemy.ext.asyncio import AsyncSession

   from app.api.deps import get_db, get_current_user
   from app.core.db import safe_db_call as _try_db  # noqa: E402
   from app.core.logging import get_logger

   logger = get_logger(__name__)
   router = APIRouter()


   @router.get("")
   async def list_params(user=Depends(get_current_user)):
       async def _q(db: AsyncSession):
           ...
       result = await _try_db(_q)
       return result or {"data": []}
   ```

4. **挂载**：在 `app/main.py` 末尾加：

   ```python
   from app.api import process_params
   app.include_router(process_params.router, prefix="/api/v1/process-params", tags=["工艺参数"])
   ```

5. **前端 API**：在 `frontend/src/services/api.ts` 增对应 export，并在页面调用。

6. **测试**：在 `backend/tests/test_<feature>.py` 至少覆盖白名单/参数校验类纯函数。

## 8. 新增 ORM 模型

模板：

```python
class ProcessParam(TimestampMixin, Base):
    __tablename__ = "process_params"

    id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
    name: Mapped[str] = mapped_column(String(200))
    line_id: Mapped[int] = mapped_column(ForeignKey("production_lines.id"))
    target: Mapped[float] = mapped_column(Float)
    upper_limit: Mapped[float] = mapped_column(Float)
    lower_limit: Mapped[float] = mapped_column(Float)
```

约定：

- 所有持久化实体统一继承 `Base + TimestampMixin`；
- 主键统一 `Integer + autoincrement=True`；
- 外键命名 `<entity>_id`；关联用 `Mapped[list["..."]] = relationship(back_populates=...)`；
- 时间字段用 `DateTime`，`server_default=func.now()`；
- 状态字段用 `String(50)` 而非 SQLAlchemy `Enum`，避免迁移噩梦（DB 端校验改用 application-level enum + check constraint）。

## 9. Neo4j 操作

### 9.1 driver 复用

`app/database.py` 在模块加载时创建一个全局 `neo4j_driver`。失败不会阻断启动；router 内部用 `_try_neo4j` 尝试 + log。

### 9.2 Cypher 安全

- **禁止**直接拼接用户输入到 Cypher 字符串；
- 优先使用 `app/api/graph.py: _TEMPLATE_WHITELIST` 列出的命名模板；
- 自由 `query` 必须经 `_assert_readonly_cypher`，凡含 `CREATE/MERGE/DELETE/SET/REMOVE/DROP/CALL/LOAD/FOREACH/USING/GRANT/REVOKE` 中任一关键字的请求一律 400。
- 标识符替换（如 `:Equipment`）必须取自服务端常量，不允许来自 body。

### 9.3 双写一致性

`app/api/ontology.py` 中 PG → Neo4j 的双写为「最佳努力」（best-effort）：异常会被记 `warning`，业务返回成功。如需强一致，考虑：

- 把 Neo4j 写入挪到事务后置的 outbox + worker；
- 或引入分布式事务（Saga 模式）。

## 10. 扩展 model-driven 动态 CRUD

新增一张可被「动态数据 CRUD」操作的表：

1. **声明白名单**：在 `app/api/_model_driven_shared.py: SAFE_COLUMNS` 加入新表 + 允许的列。
2. **声明实体映射**：如果模型名（如 `ProcessParam`）与表名 (`process_params`) 不能直接由 `.title()/.lower()` 推出，加到 `ENTITY_TABLE_MAP`。
3. **正则校验自动覆盖**：`assert_safe_identifier` 已内置 `^[a-z][a-z0-9_]*$` 检查，新增不需改动。
4. **测试**：`tests/test_model_driven_safety.py::test_safe_columns_keys_are_all_valid` 会自动覆盖新表名。

⚠ **绝对不要**在 router 里写 `f"... FROM {model_name} ..."` 式拼接。所有动态 SQL 必须复用 `model_driven_data._resolve_table` 与 `assert_safe_identifier` 的双层防御。

## 11. 代码风格与提交检查

| 项 | 工具 | 命令 |
|---|---|---|
| 语法 | `ast` | `python -m py_compile $(find app -name '*.py')` |
| 单元测试 | `pytest` | `pytest -v` |
| 类型 (可选) | `mypy` | `mypy app/` |
| 格式 (可选) | `ruff format` / `black` | 二选一，全仓库统一 |

提交前最低基线：

```bash
cd backend
python -m py_compile $(find app -name '*.py')
python -m pytest -q
```

CI 占位：建议 GitHub Actions / GitLab CI 复用上述命令。
