# ManuFoundry / Palantir Demo 0.3.4 发布说明

Release date: 2026-05-28

## 背景

0.3.4 是一次面向 SaaS 演示状态的加固版本。上一轮已经把应用页从静态示例逐步切到服务器数据，本次继续收紧多租户边界，并把动态表单、菜单、工作流、知识库和 AI Agent 相关能力整理到更接近真实产品的路径里。

## 主要变化

- 新增表单版本、租户运营字段、业务租户隔离和 SaaS 加固相关 Alembic 迁移。
- `dashboard` 与 `/program/*` 接口按当前登录用户租户过滤业务数据，避免不同租户之间串读。
- 动态表单页面读取已发布 schema，应用页可从动态表单记录生成数据视图。
- 租户管理、应用菜单、工作流申请、表单设置等前端页面同步更新。
- AI Agent 增加低代码规划与工具封装，围绕表单、菜单、页面和数据模型提供配置动作。
- 知识库、通知、规则、AI 会话等核心表补充租户字段、索引或唯一约束。

## 数据与部署要求

服务器仍使用 Docker Compose 下的 PostgreSQL、Redis、Neo4j、后端和前端服务。发布时需要执行 Alembic 迁移，并把本地开发数据库整库同步到服务器 PostgreSQL，保证以下内容和本地一致：

- 用户、角色、组织和权限配置。
- 应用菜单、动态表单、工作流配置。
- 生产、质量、设备、供应链等演示业务数据。
- AI 设置、知识库文档、知识块和对象关联数据。

GitHub 仓库只提交代码、迁移、脚本、文档和演示知识资产，不提交 `.db`、`.sqlite`、WAL 或 SHM 数据库文件。
