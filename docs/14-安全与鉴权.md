## 1. 威胁模型

| 威胁 | 严重程度 | 现状 | 文档章节 |
|---|---|---|---|
| 任意 SQL/Cypher 执行 | 高 | 双层白名单 + 只读校验 | §5, §6 |
| JWT 伪造 / 窃取 | 高 | settings.SECRET_KEY 集中签发；浏览器侧依赖 localStorage（demo 取舍） | §3, §7 |
| CSRF | 中 | 当前 `Authorization: Bearer` 头部携带，不依赖 cookie，CSRF 风险低 | §2 |
| XSS → token 泄漏 | 中 | React 默认转义；禁用 dangerouslySetInnerHTML；待迁移 HttpOnly Cookie | §7 |
| CORS 误配 → 跨域劫持 | 中 | 显式 origin 列表 + 自动禁用与 `*` 同时启用 credentials | §4 |
| 鉴权旁路（路由忘加 Depends） | 中 | `DEMO_AUTH_OPTIONAL=false` 时由 `get_current_user` 强制 401 | §2 |
| Neo4j 双写不一致 | 低 | best-effort + warning log，业务接受最终一致 | 见 `12-后端开发指南` §9.3 |
| 密码弱哈希 | 中 | 优先 bcrypt；fallback `sha256$<hex>` 显式命名空间，避免与 bcrypt 串扰 | §3 |

## 2. 鉴权流程

```
┌─ 浏览器  ───────────────────────────────────────────────
│  1. POST /api/v1/auth/login {username, password}
│  2. 收到 {token, user}，写入 localStorage
│  3. 后续请求由 axios interceptor 自动添加
│       Authorization: Bearer <token>
│  4. 401 响应触发 logout + redirect /login
└──┬───────────────────────────────────────────────────────
   │
   ▼
┌─ FastAPI  ──────────────────────────────────────────────
│  5. CORSMiddleware 校验 Origin
│  6. Depends(get_current_user) 解析 Authorization
│       ├─ 缺/坏 token & DEMO_AUTH_OPTIONAL=true → guest 主体（is_admin=False）
│       └─ 缺/坏 token & DEMO_AUTH_OPTIONAL=false → 401
│  7. Depends(get_db) → AsyncSession
│  8. Router 业务 + DB 查询
│  9. 异常 → unhandled_exception_handler → 500 (无 stacktrace)
└─────────────────────────────────────────────────────────
```

**强制鉴权切换**：生产应在 `.env` 中设：

```dotenv
DEMO_AUTH_OPTIONAL=false
SECRET_KEY=<至少 32 字节随机串>
```

如某些 router 必须公开（例如 `/health`），**显式不要**加 `Depends(get_current_user)`，已在 `app/main.py` 中以 `dependencies=[]` 方式管理（当前 demo 全部路由共享 router-level 配置，必要时按 `app.include_router(..., dependencies=[Depends(get_current_user)])` 收紧）。

## 3. JWT 与密码哈希

### 3.1 单一签发入口

```python
from app.core.security import create_access_token, decode_access_token

token = create_access_token(
    subject=user.username,
    extra={"uid": user.id, "is_admin": user.is_admin},
)
```

- 算法：`HS256`
- 密钥：`settings.SECRET_KEY`（**全仓库唯一**，旧 `app/api/auth.py` 内置 SECRET 已删除）
- 过期：`settings.ACCESS_TOKEN_EXPIRE_MINUTES`（默认 480 分钟）
- 缺 `python-jose` 时降级为 base64 JSON（仅供 demo bootstrap），日志会有 `WARNING` 提示

### 3.2 密码哈希

```python
from app.core.security import hash_password, verify_password
hashed = hash_password("S3cr3t!")
ok = verify_password("S3cr3t!", hashed)
```

- 默认 `passlib[bcrypt]`（cost = passlib 默认）
- fallback 输出固定前缀 `sha256$<hex>`，`verify_password` 据此分支选算法
- ⚠ 不要直接对密码做 `hashlib.sha256().hexdigest()` —— 必须经 `core.security`；`admin.py::_hash_password` 已改为委派至此

### 3.3 token 测试

```bash
cd backend
python -m pytest tests/test_security.py -v
```

覆盖：

- 正常往返；
- 篡改签名（依赖 python-jose）；
- 垃圾字符串拒绝；
- 密码哈希命名空间不重叠。

## 4. CORS 配置

`app/main.py`：

```python
_cors_origins = settings.CORS_ORIGINS
_allow_credentials = "*" not in _cors_origins

app.add_middleware(
    CORSMiddleware,
    allow_origins=_cors_origins,
    allow_credentials=_allow_credentials,
    allow_methods=["*"],
    allow_headers=["*"],
    expose_headers=["Content-Disposition"],
)
```

- 默认 `["http://localhost:3000", "http://localhost:5173", "http://127.0.0.1:3000"]`；
- `*` + `credentials=True` 是浏览器明确禁止的组合；本实现自动让二者互斥；
- 生产可在 `.env` 中改：

  ```dotenv
  CORS_ORIGINS=["https://manufoundry.example.com","https://admin.manufoundry.example.com"]
  ```

## 5. SQL 注入防御

### 5.1 静态查询

所有 SQLAlchemy ORM 查询自动参数化，不需特殊处理。

### 5.2 动态表/列名（`model_driven_data`）

`app/api/model_driven_data.py` 和 `_model_driven_shared.py` 实现**双层防御**：

- **白名单**：只有 `SAFE_COLUMNS` 内的表名 + 列名才会被拼入 SQL；
- **正则**：`assert_safe_identifier(name)` 用 `^[a-z][a-z0-9_]*$` 阻止任何含分号、空格、引号、大写、点号的标识符；
- **参数化值**：`text(sql)` + `:bind` 绑定值。

测试覆盖：

```bash
pytest tests/test_model_driven_safety.py -v
```

新加表时只需扩展 `SAFE_COLUMNS`，正则会自动覆盖（`test_safe_columns_keys_are_all_valid` 强制保证）。

### 5.3 日志中的 SQL

`echo=settings.DEBUG`，生产 (`DEBUG=false`) 不打印 SQL；调试时再切换。

## 6. Cypher 注入防御

### 6.1 白名单模板

`app/api/graph.py: _TEMPLATE_WHITELIST`：

```python
{
  "stats":         "MATCH (n) RETURN labels(n)[0] AS label, count(*) ...",
  "all_factories": "MATCH (n:Factory) RETURN n LIMIT $limit",
  "neighbors":     "MATCH (n {pg_id: $entity_id})-[r]-(m) RETURN ...",
}
```

前端首选传 `{template, params}`，服务端取常量字符串，body 仅供绑定参数。

### 6.2 自由 query 的只读校验

```python
_FORBIDDEN_CYPHER_PATTERN = re.compile(
    r"\b(CREATE|MERGE|DELETE|DETACH|SET|REMOVE|DROP|CALL|LOAD|FOREACH|USING|GRANT|REVOKE)\b",
    re.IGNORECASE,
)
```

凡命中任一关键字 → 400。允许 `MATCH/RETURN/WHERE/WITH/UNWIND/LIMIT/ORDER/SKIP`。

### 6.3 标签拼接的安全示例

`graph_service.py::create_entity` 仍使用 f-string 拼标签，但前置了 `safe_labels` 白名单，保证 label 来自已知 enum。同样模式可推广到任何需要标识符替换的场景：

```python
SAFE_LABELS = {"Factory", "Equipment", ...}
if label not in SAFE_LABELS:
    raise ValueError(...)
cypher = f"CREATE (n:{label} {{...}})"
```

### 6.4 测试

```bash
pytest tests/test_graph_cypher_safety.py -v
```

包含：

- `MATCH/RETURN/UNWIND` 必通过；
- `CREATE/DETACH DELETE/SET/MERGE/DROP/CALL/LOAD/FOREACH` 必拒绝；
- 自带的所有模板自身必须通过只读校验（自一致）。

## 7. 前端安全

| 项 | 当前 | 备注 |
|---|---|---|
| token 存储 | `localStorage('mf_token')` | XSS 即可窃取；后续目标改 HttpOnly Cookie + CSRF Token |
| 自动注入 | axios `request` interceptor | 头部 `Authorization: Bearer <token>` |
| 401 处理 | axios `response` interceptor | 清空 + 跳 `/login`；防止重定向死循环（已检测 `pathname.startsWith('/login')`） |
| HTML 注入 | React 默认转义 | 严禁 `dangerouslySetInnerHTML`；如必须用，只接受白名单标签的 sanitize 后内容（推荐 `DOMPurify`） |
| 第三方脚本 | 当前未加载 | 后续如接 GA/Sentry，注意 CSP |

加固路线（按优先级）：

1. 后端开启 HttpOnly Cookie 模式：`/auth/login` 在 `Set-Cookie` 中写 token，`get_current_user` 兼容 Cookie 与 Header 两种来源；
2. 引入 CSP header（FastAPI middleware 或 nginx `add_header Content-Security-Policy ...`）；
3. 接入 SRI（subresource integrity）后再上 CDN。
