## 1. 背景与范围

仓库初始状态（审计前）存在多处生产级风险与工程缺陷，主要包括：

- 鉴权链路缺失（前端不附带 token、后端无依赖检查）
- `auth.py` 内置 `SECRET_KEY` 与 `settings.SECRET_KEY` 不同步
- CORS `*` + `credentials=True` 错误组合
- `_try_db` 静默吞错在 13 个 router 重复
- `model_driven.py` 643 行单文件 + 动态 SQL 标识符无正则校验
- TS 构建脚本 `tsc -b` 与单一 `tsconfig.json` 不兼容
- Docker compose 顶部 `version: "3.8"` 已废弃；frontend volume 错挂到 nginx 镜像
- alembic 缺 versions/、与 `init_db` 重复机制
- 缺少 `.gitignore` / `.env.example` / `.dockerignore`
- 0 单元测试

本次审计在不破坏 demo 启动行为的前提下完成 P0~P3 改造，所有变更均通过 AST 解析（48 个 Python 文件全部 OK）。

## 2. P0 安全 / 鉴权改造

### 2.1 新增模块

| 文件 | 作用 |
|---|---|
| `backend/app/core/__init__.py` | 包初始化 |
| `backend/app/core/logging.py` | `setup_logging` / `get_logger` 集中入口 |
| `backend/app/core/security.py` | `create_access_token` / `decode_access_token` / `hash_password` / `verify_password`，**全局唯一 SECRET_KEY 出口** |
| `backend/app/core/db.py` | `db_session()` async ctx + `safe_db_call()` 兼容层 |
| `backend/app/api/deps.py` | `get_current_user` (Bearer + legacy query)、`get_db`、`require_admin` |

### 2.2 鉴权链路

- `frontend/src/services/api.ts`：新增 axios `request` / `response` 拦截器，自动注入 `Authorization: Bearer <token>`，401 时清空登录态并跳 `/login`，带防跳转死循环。
- `frontend/src/App.tsx`：散落的 5 处 `fetch('/api/v1/...')` 改为统一调用 `services/api.ts` 暴露的函数；`Modal.confirm` 中 `document.getElementById` 反 React 模式改为 `React.createRef` + 受控读值。
- `backend/app/api/auth.py`：删除内置 `SECRET_KEY = "..."` 与 `_create_token/_verify_token/_hash_password/_check_password` 内联实现，全部委派 `core.security`；`/me` 改为 `Depends(get_current_user)`，从 Authorization Header 解析。
- `backend/app/main.py`：CORS 改为读 `settings.CORS_ORIGINS`，自动检测 `*` 与 credentials 互斥；新增全局 `Exception` handler 返回 500 + 类型名（隐藏 stacktrace）。

### 2.3 防注入

- `backend/app/api/graph.py`：新增 `_TEMPLATE_WHITELIST` + `_assert_readonly_cypher`；`/query` 端点要求传 `template` 名或自由 `query`，自由 query 必须通过只读关键字校验。
- `backend/app/api/_model_driven_shared.py` & `model_driven_data.py`：所有动态 SQL 入口经 `_resolve_table` → 白名单 + `assert_safe_identifier(^[a-z][a-z0-9_]*$)` 双层防御；列名同样校验。

### 2.4 配置

- `backend/app/config.py`：新增 `DEMO_AUTH_OPTIONAL` / `CORS_ORIGINS` / `LOG_LEVEL`；`SECRET_KEY` 集中字段。
- `backend/app/database.py`：所有 `print` 替换为 `logger`；PG 引擎初始化失败不再阻塞模块导入。

## 3. P1 架构重整

### 3.1 日志统一

`app.core.logging.setup_logging()` 在 `app.main` 启动时调用一次；所有 router 通过 `get_logger(__name__)` 拿到带模块名的 logger，告别 `print`。

### 3.2 DB 会话统一

`safe_db_call` 提供与旧 `_try_db` 完全兼容的签名（`async def fn(session)`），失败时 rollback + log + 返回 None。详细见 §5.2。

### 3.3 TypeScript 构建

| 文件 | 改动 |
|---|---|
| `frontend/tsconfig.json` | 改为 `references` 元配置 |
| `frontend/tsconfig.app.json` | 应用代码 TS 配置（`composite: true`） |
| `frontend/tsconfig.node.json` | `vite.config.ts` 等 node 侧配置 |
| `frontend/package.json` | 新增 `type-check` 脚本 |

`tsc -b` 现在能正确按引用顺序构建。

## 5. P3 代码质量

### 5.1 model_driven.py 拆分

| 文件 | 行数 | 职责 |
|---|---|---|
| `_model_driven_shared.py` | ~215 | Pydantic 输入 schemas、`SAFE_COLUMNS` / `ENTITY_TABLE_MAP`、3 套 mock、`assert_safe_identifier`、`try_db = safe_db_call` |
| `model_driven_meta.py` | ~250 | 元模型 CRUD + 字段 + 本体导入 + 页面配置 |
| `model_driven_data.py` | ~125 | 动态数据 CRUD + `_resolve_table` 双层防御 |
| `model_driven_menus.py` | ~95 | 菜单项 CRUD |
| `model_driven.py` | 23 | `APIRouter().include_router(...)` 聚合，对外保留单一 `router` 符号 |

### 5.2 _try_db 收敛

- 12 个 router（admin / ai_assistant / analytics / dashboard / data_sources / maintenance / ontology / pipeline / quality / reports / supply_chain / workflow）的本地 `_try_db` 实现全部删除；
- 改为 `from app.core.db import safe_db_call as _try_db`，调用点（`await _try_db(_query)`）零修改；
- `_model_driven_shared.try_db` 同步重 export 自 `safe_db_call`，全局只剩一份实现。

副作用清理：

- `admin.py::_hash_password` 改为委派 `core.security.hash_password`，与 `auth.py` 共用算法（避免 admin 写出的密码被 auth 验签失败）；
- `ontology.py` 中 3 处 Neo4j 双写的 `except: pass` 改为带 logger 的告警，体现「best-effort + 可观测」语义。

### 5.3 配置外移

- `frontend/src/config/menus.ts`：从 `App.tsx` 抽出 `BUSINESS_MENUS / ROLE_MENU_MAP / BREADCRUMB_MAP / APPROVAL_STATUS_CONFIG`，新增菜单/角色/状态变更只需改这一个文件。
- `backend/app/models/graph_models.py`：移除未使用 `dataclass` import。

### 5.4 测试基线

| 文件 | 用例 |
|---|---|
| `tests/conftest.py` | `sys.path` 注入 + 测试环境变量 |
| `tests/test_security.py` | JWT 往返 / tampered / hash 命名空间，5 用例 |
| `tests/test_graph_cypher_safety.py` | 只读 Cypher 校验 + 模板自一致，9 用例 |
| `tests/test_model_driven_safety.py` | 标识符正则 / 表名决议 / 注入拒绝，6 用例 |
| `tests/test_logging_setup.py` | logger 幂等 / namespace，2 用例 |

冒烟运行（无需 pytest 的 fallback 路径）实测：6/6 通过。

## 6. 验证与回归风险

### 6.1 验证

- AST 解析：48 个 Python 文件全部通过；
- 冒烟测试：6 用例通过（`test_security` + `test_logging_setup`）；
- 拓扑校验：`grep` 确认 12 个 router 全部 `from app.core.db import safe_db_call`，`_model_driven_shared` 也已 re-export；
- `model_driven.py` 仅 23 行，符合「聚合入口」语义。

### 6.2 回归风险

| 改动 | 潜在回归 | 缓解 |
|---|---|---|
| `auth.py` 改为 Bearer 头 | 旧前端调用 `/auth/me?token=...` 会被 `get_current_user` 兼容（保留 query 路径） | 已保留兼容 |
| CORS 严格化 | 跨域域名未列入 `CORS_ORIGINS` 时浏览器拦截 | 提供 `.env.example` 显式声明 |
| `DEMO_AUTH_OPTIONAL=true`（默认） | 生产忘记关闭会让缺 token 退化为 guest | 明确文档化 + 安全 checklist |
| graph `/query` 拒绝写操作 | 如果有页面依赖 `MERGE/CREATE` 调用 → 400 | 文档明示，迁移到 template 模式 |
| `_try_db` 改 `safe_db_call` | 行为等价（rollback + log + None），但日志多了 warning 行 | 调 LOG_LEVEL 即可减噪 |
| `model_driven.py` 拆分 | 外部 `from app.api.model_driven import xxx` 引用断裂 | `model_driven.router` 仍是唯一对外符号；其他符号不应被外部直接导入 |

### 6.3 容易踩坑

- 文件尾 NUL padding：本次过程中发现 13 个 router 文件被中间环节填充了 NUL 字节，已批量截断；后续若再次出现 `python -m py_compile` 报 `null bytes`，先用 `tr -d '\0'` 清理。

## 7. 遗留事项

按优先级降序：

1. **生产 SECRET_KEY 流转**：当前 `.env.example` 仍是 demo 字符串，需要接入 vault / k8s secret，提供轮转机制；
2. **token 改 HttpOnly Cookie**：消除 localStorage 的 XSS 暴露面，详见 `14-安全与鉴权.md §7`；
3. **rate limit**：登录、AI、graph query 等接口需要限流（`fastapi-limiter` 或 nginx）；
4. **mock 全局状态加锁**：`auth._MOCK_USERS` / `workflow._MOCK_INSTANCES` / `_model_driven_shared.MOCK_*` 仍是普通 list/dict，并发污染风险（demo 可接受）；
5. **集成测试**：`pytest-asyncio + httpx.AsyncClient` 跑通 `/auth/login` → `/dashboard/overview` 全链路；
6. **前端 vitest 接入**：`authStore` 与 `services/api.ts` 拦截器写最少 5 用例；
7. **AI Assistant 真接入**：`requirements.txt` 已注释化 `langchain*`，激活时同步在 `services/ai/` 增加适配；
8. **Pipeline 真执行**：`pipeline_engine.py` 当前是模拟，可接 Celery / Dramatiq；
9. **k8s helm chart**：当前仅有 docker-compose，K8s 部署待补；
10. **可观测性**：Prometheus 指标、OpenTelemetry trace、Loki 日志聚合。
