## 模式一：快速体验（SQLite 零依赖）

最简单的启动方式，无需安装任何数据库。后端自动检测环境，降级使用 SQLite，并在首次启动时自动创建表结构和导入模拟数据。

### 前置条件

- Python 3.11+
- Node.js 18+

### 步骤一：启动后端

```bash
cd backend
pip install -r requirements.txt
python -m uvicorn app.main:app --port 8000
```

启动成功后会看到如下日志：

```
[ManuFoundry] SQLite database initialized: C:\...\backend\manufoundry.db
[ManuFoundry] Seeding database...
[ManuFoundry] Seeded 2XXX rows total
INFO:     Uvicorn running on http://127.0.0.1:8000
```

> **说明**：首次启动需要导入模拟数据，耗时约 5-10 秒。后续启动会跳过数据导入（检测到数据已存在）。

### 步骤二：启动前端

打开新的终端窗口：

```bash
cd frontend
npm install
npm run dev
```

启动成功后访问 http://localhost:3000。

### SQLite 模式说明

| 特性 | 说明 |
|------|------|
| 数据库文件 | `backend/manufoundry.db`（自动创建） |
| Neo4j | 自动跳过，图谱相关 API 返回 Mock 数据 |
| Redis | 自动跳过，不使用缓存 |
| 数据完整性 | 全部模拟数据正常导入 |
| 适用场景 | 快速体验、演示、开发调试 |

如需重置数据，删除 `manufoundry.db` 文件后重启后端即可。

## 模式二：开发模式（Docker Compose）

使用 Docker Compose 启动完整的 PostgreSQL + Neo4j + Redis 技术栈，适合需要完整功能（知识图谱、缓存等）的开发场景。

### 前置条件

- Docker 20.10+
- Docker Compose v2+

### 步骤一：创建环境变量文件

在 `backend/` 目录下创建 `.env` 文件：

```bash
# backend/.env
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_USER=manufoundry
POSTGRES_PASSWORD=manufoundry123
POSTGRES_DB=manufoundry

NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=manufoundry123

REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=0

DEBUG=true
```

### 步骤二：启动基础设施

```bash
cd docker
docker-compose up -d postgres neo4j redis
```

等待所有服务健康检查通过：

```bash
docker-compose ps
```

确认三个服务状态均为 `healthy`。

### 步骤三：导入种子数据

```bash
cd scripts
python seed_data.py
```

脚本执行流程：
1. 创建全部数据库表
2. 导入 16 个 JSON 种子文件到 PostgreSQL
3. 构建 Neo4j 知识图谱

### 步骤四：启动后端和前端

```bash
# 终端 1：后端
cd backend
pip install -r requirements.txt
python -m uvicorn app.main:app --reload --port 8000

# 终端 2：前端
cd frontend
npm install
npm run dev
```

### 一键启动（Docker 全容器）

也可以使用 Docker Compose 直接启动所有服务（包含后端和前端容器）：

```bash
cd docker
docker-compose up -d
```

这会启动 5 个容器：
- `manufoundry-postgres` (PostgreSQL 16)
- `manufoundry-neo4j` (Neo4j 5)
- `manufoundry-redis` (Redis 7)
- `manufoundry-backend` (FastAPI)
- `manufoundry-frontend` (React + Nginx)

## 模式三：生产部署

### PostgreSQL 16 配置

```bash
# 安装 PostgreSQL 16
# Ubuntu
sudo apt install postgresql-16

# 创建数据库和用户
sudo -u postgres psql
CREATE USER manufoundry WITH PASSWORD 'your_secure_password';
CREATE DATABASE manufoundry OWNER manufoundry;
GRANT ALL PRIVILEGES ON DATABASE manufoundry TO manufoundry;
```

**`postgresql.conf` 优化建议**：

```ini
# 连接配置
max_connections = 100
listen_addresses = '*'

# 内存配置（根据服务器内存调整）
shared_buffers = 256MB
effective_cache_size = 768MB
work_mem = 4MB

# WAL 配置
wal_level = replica
max_wal_size = 1GB
```

### Neo4j 5 配置

```bash
# 安装 Neo4j 5 Community
# 参考 https://neo4j.com/docs/operations-manual/current/installation/

# 配置 neo4j.conf
server.default_listen_address=0.0.0.0
server.bolt.listen_address=0.0.0.0:7687
server.http.listen_address=0.0.0.0:7474

# 内存配置
server.memory.heap.initial_size=512m
server.memory.heap.max_size=1G
server.memory.pagecache.size=256m
```

### Redis 7 配置

```bash
# 安装 Redis 7
sudo apt install redis-server

# /etc/redis/redis.conf
bind 127.0.0.1
port 6379
maxmemory 256mb
maxmemory-policy allkeys-lru
```

### Nginx 反向代理配置

```nginx
server {
    listen 80;
    server_name manufoundry.example.com;

    # 前端静态文件
    location / {
        root /opt/manufoundry/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
        proxy_connect_timeout 75s;
    }

    # Swagger 文档
    location /docs {
        proxy_pass http://127.0.0.1:8000/docs;
    }

    location /openapi.json {
        proxy_pass http://127.0.0.1:8000/openapi.json;
    }
}
```

### Systemd 服务文件

后端服务 `manufoundry-backend.service`：

```ini
[Unit]
Description=ManuFoundry Backend API
After=network.target postgresql.service neo4j.service redis.service

[Service]
Type=simple
User=manufoundry
Group=manufoundry
WorkingDirectory=/opt/manufoundry/backend
Environment="PATH=/opt/manufoundry/backend/venv/bin"
ExecStart=/opt/manufoundry/backend/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl enable manufoundry-backend
sudo systemctl start manufoundry-backend
```

## 环境变量配置

全部环境变量通过 `backend/.env` 文件或系统环境变量配置。未设置时使用默认值。

### 数据库配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `POSTGRES_HOST` | `localhost` | PostgreSQL 主机地址 |
| `POSTGRES_PORT` | `5432` | PostgreSQL 端口 |
| `POSTGRES_USER` | `manufoundry` | 数据库用户名 |
| `POSTGRES_PASSWORD` | `manufoundry123` | 数据库密码（生产环境务必修改） |
| `POSTGRES_DB` | `manufoundry` | 数据库名称 |

> **自动降级**：如果 PostgreSQL 连接失败（如未安装 asyncpg 或连接被拒绝），系统自动降级为 SQLite 模式，无需额外配置。

### Neo4j 配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `NEO4J_URI` | `bolt://localhost:7687` | Neo4j Bolt 协议地址 |
| `NEO4J_USER` | `neo4j` | Neo4j 用户名 |
| `NEO4J_PASSWORD` | `manufoundry123` | Neo4j 密码 |

> **自动降级**：如果 Neo4j 连接失败，图谱相关 API 自动返回 Mock 数据，不影响其他功能。

### Redis 配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `REDIS_HOST` | `localhost` | Redis 主机地址 |
| `REDIS_PORT` | `6379` | Redis 端口 |
| `REDIS_DB` | `0` | Redis 数据库编号 |

### 应用配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `DEBUG` | `true` | 调试模式（开启 SQL 日志） |
| `SECRET_KEY` | `manufoundry-secret-key-...` | JWT 签名密钥（生产环境务必修改） |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | `1440` | Token 过期时间（分钟，默认 24 小时） |

### AI 配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `OPENAI_API_KEY` | 空 | OpenAI API Key（AI 助手功能需要） |
| `OPENAI_MODEL` | `gpt-4o` | 使用的 OpenAI 模型名称 |

> AI 助手功能为可选，未配置 API Key 时返回 Mock 响应。

### 组合连接 URL

系统自动根据上述配置生成连接 URL，无需手动设置：

```
DATABASE_URL       = postgresql+asyncpg://user:pass@host:port/db
DATABASE_URL_SYNC  = postgresql+psycopg2://user:pass@host:port/db
```

## 数据库初始化

### 自动迁移（SQLite 模式）

SQLite 模式下，后端启动时自动执行：
1. 创建所有表结构（`Base.metadata.create_all`）
2. 检测是否为空库
3. 自动导入种子数据

无需手动操作。

### 手动 Alembic 迁移（PostgreSQL 模式）

```bash
cd backend

# 生成迁移文件
alembic revision --autogenerate -m "描述信息"

# 执行迁移
alembic upgrade head

# 查看当前版本
alembic current

# 回退一个版本
alembic downgrade -1
```

### 种子数据导入

**方式一：完整导入脚本（PostgreSQL + Neo4j）**

```bash
cd scripts
python seed_data.py
```

该脚本会：
1. 根据 SQLAlchemy 模型创建全部表
2. 从 `data/seed/*.json` 导入数据到 PostgreSQL（16 个文件）
3. 从种子数据构建 Neo4j 知识图谱

**方式二：重新生成模拟数据**

```bash
cd data/simulators
python generate_sample_data.py
```

该脚本会重新生成所有 `data/seed/*.json` 文件，然后使用 `seed_data.py` 导入。

## 常见问题排查

### 端口 8000 已被占用

**现象**：启动后端时报错 `Address already in use`。

**解决方案**：

```bash
# 方案 1：更换端口
python -m uvicorn app.main:app --port 8001

# 方案 2：查找并终止占用进程
# Linux/macOS
lsof -i :8000
kill -9 <PID>

# Windows
netstat -ano | findstr :8000
taskkill /PID <PID> /F
```

如果更换了后端端口，前端需要同步修改 `vite.config.ts` 中的代理目标：

```typescript
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8001',  // 修改为新端口
      changeOrigin: true,
    },
  },
},
```

### SQLite 与 PostgreSQL 自动检测

**工作原理**：

系统通过尝试 `import asyncpg` 判断是否使用 PostgreSQL：
- 成功导入 -> 使用 PostgreSQL（`DB_TYPE = "postgresql"`）
- 导入失败 -> 自动降级为 SQLite（`DB_TYPE = "sqlite"`）

SQLite 模式下：
- 数据库文件位于 `backend/manufoundry.db`
- 首次启动自动建表并导入数据
- 不支持并发写入（适合单用户开发）

### Neo4j 连接被拒绝（自动降级为 Mock）

**现象**：日志中无 Neo4j 错误，但图谱相关 API 返回的是固定 Mock 数据。

**原因**：Neo4j 服务未启动或连接配置不正确。

**检查步骤**：

```bash
# 1. 检查 Neo4j 是否运行
curl http://localhost:7474

# 2. 测试 Bolt 连接
python -c "from neo4j import GraphDatabase; d=GraphDatabase.driver('bolt://localhost:7687',auth=('neo4j','manufoundry123')); print('OK')"

# 3. 如果使用 Docker，检查容器状态
docker ps | grep neo4j
```

> **说明**：Neo4j 为可选组件，连接失败不影响系统其他功能。图谱相关 API 会自动返回 Mock 数据。

### 前端代理配置问题

**现象**：前端页面加载正常，但 API 请求返回 404 或连接被拒绝。

**检查项**：

1. 确认后端已启动并监听 8000 端口
2. 检查 `frontend/vite.config.ts` 中 proxy 配置：

```typescript
server: {
  port: 3000,
  proxy: {
    '/api': {
      target: 'http://localhost:8000',
      changeOrigin: true,
    },
  },
},
```

3. 确认使用 `npm run dev`（Vite 开发服务器），而非 `npm run preview`

### 中文 Windows 编码问题

**现象**：导入种子数据时出现 `UnicodeDecodeError` 或中文显示乱码。

**解决方案**：

```bash
# 方案 1：设置终端编码
set PYTHONIOENCODING=utf-8
chcp 65001

# 方案 2：在 Python 脚本中已使用 encoding="utf-8" 打开文件
# seed_data.py 和 database.py 中已做处理
```

种子数据文件（`data/seed/*.json`）均使用 UTF-8 编码，导入代码中已指定 `encoding="utf-8"`。

### pip install 安装失败

**常见原因及解决方案**：

| 包名 | 问题 | 解决方案 |
|------|------|---------|
| `asyncpg` | 需要 C 编译器 | Windows: 安装 Visual Studio Build Tools；或跳过（降级 SQLite） |
| `psycopg2-binary` | 需要 pg_config | 安装 PostgreSQL 开发库；或跳过（降级 SQLite） |
| `prophet` | 依赖 pystan | 先 `pip install pystan`，再安装 prophet |
| `neo4j` | 通常无问题 | 确保网络可访问 PyPI |

**最小依赖模式**（仅需 FastAPI + SQLAlchemy + aiosqlite）：

如果只想快速体验 SQLite 模式，可以创建简化的 requirements 文件：

```
fastapi
uvicorn[standard]
sqlalchemy
aiosqlite
pydantic
pydantic-settings
```

## 健康检查

### 后端 API 健康检查

```bash
# 基础健康检查
curl http://localhost:8000/health
# 预期返回：{"status": "healthy"}

# 运营总览（验证数据是否加载）
curl http://localhost:8000/api/v1/dashboard/overview
# 预期返回：包含 factories、equipment、work_orders 等字段

# 应用信息
curl http://localhost:8000/
# 预期返回：{"name": "ManuFoundry", "version": "0.1.0", ...}
```

### 前端健康检查

```bash
# 直接访问
curl http://localhost:3000
# 预期返回：HTML 页面内容
```

### Swagger API 文档

浏览器访问：

- http://localhost:8000/docs -- Swagger UI 交互式文档
- http://localhost:8000/redoc -- ReDoc 文档

在 Swagger UI 中可以直接测试所有 API 端点。

### Docker 容器健康检查

```bash
cd docker

# 查看所有容器状态
docker-compose ps

# 预期输出：
# manufoundry-postgres   running (healthy)
# manufoundry-neo4j      running (healthy)
# manufoundry-redis      running (healthy)
# manufoundry-backend    running
# manufoundry-frontend   running

# 查看后端日志
docker-compose logs -f backend

# 查看 PostgreSQL 连接
docker-compose exec postgres pg_isready -U manufoundry

# 查看 Neo4j 状态
curl http://localhost:7474

# 查看 Redis 连接
docker-compose exec redis redis-cli ping
# 预期返回：PONG
```

### 数据完整性验证

```bash
# 通过 API 验证各模块数据
curl http://localhost:8000/api/v1/dashboard/overview
curl http://localhost:8000/api/v1/ontology/entities/Factory/instances
curl http://localhost:8000/api/v1/data-sources
```

验证要点：
- `overview` 返回 3 个工厂、65 台设备、48 个工单
- `Factory/instances` 返回 3 条工厂记录
- `data-sources` 返回 8 个数据源（Mock 模式）
